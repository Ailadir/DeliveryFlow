# HappyPhone

Веб-приложение для оформления заявок на доставку посылок. Включает многошаговую форму создания заявки, историю заявок с поиском и фильтрацией, а также детальный просмотр каждой заявки. Все данные хранятся в браузере — бэкенд не требуется.

---

## Маршруты

### `/` — Создание заявки

Многошаговая форма из трёх шагов. Черновик автоматически сохраняется в `localStorage` — при перезагрузке страницы данные не теряются.

**Шаг 1 — Отправитель**
- Имя, телефон, город
- Поле телефона маскирует ввод в реальном времени: `+7 (XXX) XXX-XX-XX`
- Принимаются только цифры; ввод с `8` или `+7` нормализуется автоматически

**Шаг 2 — Получатель и груз**
- Имя, город, тип груза (документы / хрупкое / обычный груз), вес (0.1–30 кг)
- Кросс-полевая валидация: город получателя не должен совпадать с городом отправителя

**Шаг 3 — Подтверждение**
- Просмотр введённых данных перед отправкой
- Чекбокс принятия условий
- При отправке создаётся заявка, черновик очищается, пользователь перенаправляется в историю

### `/orders` — История заявок

- Список всех созданных заявок в виде карточек (1–3 колонки в зависимости от экрана)
- **Поиск** — по имени/городу отправителя, имени/городу получателя, номеру телефона (с дебаунсом 300 мс)
- **Фильтрация** — по типу груза, мульти-выбор
- Удаление заявки с диалогом подтверждения
- Кнопка быстрого создания новой заявки

### `/orders/[id]` — Детали заявки

- Полная информация по конкретной заявке
- Статусный бейдж с цветовой кодировкой
- Форматированный номер телефона и дата создания

## Хранение данных

Все данные хранятся в `localStorage` — никакого бэкенда не требуется.

| Ключ | Тип | Содержимое |
|---|---|---|
| `happyphone_orders` | `Order[]` | Список всех заявок |
| `happyphone_form_draft` | `FormDraft` | Черновик текущей формы |
| `happyphone_theme` | `'light' \| 'dark' \| 'system'` | Выбранная тема |

---

## Статусы заявок

| Статус | Цвет | Описание |
|---|---|---|
| `PENDING` | Жёлтый | Ожидает обработки |
| `PROCESSING` | Синий | В обработке |
| `COLLECTING` | Фиолетовый | Собирается |
| `READY_TO_SHIP` | Индиго | Готов к отправке |
| `IN_TRANSIT` | Оранжевый | В пути |
| `DELIVERED` | Зелёный | Доставлена |
| `CANCELLED` | Красный | Отменена |

Все заявки, созданные через форму, получают статус `PENDING`. Изменение статуса в текущей версии не предусмотрено.

## Тёмная тема

- Переключение через кнопку в шапке (солнце / луна)
- Класс `dark` применяется к `<html>`
- Инлайн-скрипт в `layout.tsx` читает `localStorage` до гидрации React — исключает мерцание (FOUC)
- Tailwind v4: тёмный вариант настроен через `@custom-variant dark` в `globals.css`, без `tailwind.config.ts`

---

## Валидация

Используется **Zod v4**. Схемы находятся в `src/lib/validation.ts`, сообщения об ошибках на русском языке.

- `step1Schema` — имя (мин. 2 символа), телефон (regex `+7 (XXX) XXX-XX-XX`), город
- `step2Schema` — получатель, город, тип груза, вес + кросс-валидация городов
- `step3Schema` — обязательное принятие условий

---

## Адаптивность

Приложение оптимизировано для мобильных и десктопных экранов:

- Мобильные (`< 640px`): бургер-меню, сокращённые подписи шагов, компактные кнопки
- Десктоп (`≥ 640px`): полная навигация в шапке, полные подписи
